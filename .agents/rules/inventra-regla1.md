# Regla 01 — Contexto del producto Inventra

## Identidad del proyecto
**Nombre:** Inventra  
**Tipo:** Panel administrativo con inventario  
**Plataformas iniciales:** Android + Windows + landing web pública  
**Objetivo:** Construir un producto multiplataforma serio, mantenible y escalable, apto para portafolio y posible uso real.

## Qué es Inventra
Inventra es una solución para:
- gestión de productos
- categorías
- stock
- movimientos de inventario
- usuarios
- roles
- reportes básicos
- configuración operativa

No es solo una lista de productos.  
Debe sentirse como un **panel administrativo moderno con inventario**.

## Usuarios objetivo
- pequeños negocios
- tiendas
- negocios familiares
- restaurantes pequeños
- emprendimientos con almacén
- usuarios que hoy usan Excel o cuadernos

## Roles base
- **Administrador:** configura, gestiona usuarios, productos, reportes y ajustes
- **Supervisor:** revisa movimientos, consulta reportes, controla operación
- **Operador:** registra entradas/salidas y consulta inventario

## Objetivos del MVP
El MVP debe incluir:
- login
- dashboard base
- productos
- categorías
- stock
- movimientos
- usuarios básicos
- configuración mínima
- landing web pública

## Objetivos técnicos
El proyecto debe:
- verse profesional
- compilar y mantenerse limpio
- crecer sin romperse
- tener arquitectura clara
- separar UI, lógica y datos
- funcionar bien tanto en móvil como en escritorio

## Restricciones
No improvisar:
- estructuras desordenadas
- pantallas gigantes
- lógica de negocio dentro de widgets
- consultas directas a Supabase dentro de la UI
- decisiones visuales inconsistentes

## Meta del proyecto
La meta no es solo “que funcione”.
La meta es construir una base profesional, clara y escalable que pueda:
- usarse
- mostrarse
- extenderse
- mantenerse con orden
